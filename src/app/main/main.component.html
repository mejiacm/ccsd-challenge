SWAPI Challenge

Your challenged to build a simple applcations that queries the SWAPI API. This is a fun API that displays Star Wars movie franchise data. Please don't use ay wrapper/library, we want to see your coding skills!

The challange consists on the following:
  • To start please fork this base project on StackBlitz
  • Displaying data using the routes on the left menu by querying the following endpoints:
            
            {
                "films": "https://swapi.co/api/films/",
                "people": "https://swapi.co/api/people/",
                "planets": "https://swapi.co/api/planets/",
                "species": "https://swapi.co/api/species/",
                "starships": "https://swapi.co/api/starships/",
                "vehicles": "https://swapi.co/api/vehicles/"
            }
    For instance, the "Films" route on the left should display data on an HTML table (suggested) from the
    https://swapi.co/api/films/
    endpoint and the result count should be displayed in the Total Results green square at the top right corner. You don't have to display all columns, with just a few is enough since we are just trying to assess your ability to work with an API response.
  • Create a NodeJS/Express API for login (we don't use Firebase)
  • Properly secure the application by adding a login component
Once you are done please send us the link to your deployed app and code, the team will then meet to check your code. Please keep in mind that we need to be able to both, see your code and see the app running. Check the resources section below for some suggestions. If you have any questions please let us know.

Bonus Points!

This is totally optional but you will earn bonus points if you complete any/all of the following:
  • Link data internally. ie: If you query the "people" endpoint the first results is "Luke Skywalker", on the data returned you can link the "starships" to the route at "https://swapi.co/api/starships/" and filter it displaying only data for the starships 12 & 22 which are related to the pilot.
  • Add a search box on any/all the routes to filter data
  • Use Boostrap components and apply styles at your discretion
  • Make the "Toggle Menu" button work, use Angular animations if possible
  • If you feel like showing off your skills adding some fancy features we would love to see that too!
